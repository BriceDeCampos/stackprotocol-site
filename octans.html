Deux environnements, un seul moteur

Octans s'installe sur votre machine. Vous choisissez comment vous l'utilises.

Le moteur est le même dans les deux cas. Vous gardez la main sur le confort que vous voulez y mettre. Aucun choix n'est irréversible.

Mode efficace

VSCode + Claude Code

L'installation minimale. Tout marche, rien de superflu.

  • Le moteur Octans tourne intégralement
  • Édition Markdown directe dans VSCode
  • Terminal intégré, Git, debug, extensions familières
  • Idéal si vous codes déjà ou si vous voulez la stack la plus légère
  • Aucun setup Obsidian à apprendre

Pour qui : développeurs, profils technique, utilisateurs qui veulent une stack minimale et un workflow code-centric.

Recommandé · mode confort

VSCode + Claude Code + Obsidian

La même installation, avec un explorateur de connaissances par-dessus.

  • Tout ce que fait VSCode + Claude Code
  • Vue du graphe — vous voyez vos notes se relier
  • Liens cliquables entre les notes (wikilinks)
  • Requêtes sur vos notes (Dataview) et plugins riches
  • Édition Markdown plus douce, palette de commandes, recherche avancée

Pour qui : porteurs de projet, indépendants, étudiants, entrepreneurs — tous ceux qui pensent en notes reliées plus qu'en lignes de code.

  • Aucun appel cloud caché. Le moteur tourne en local sur votre machine. Pas de serveur intermédiaire entre votre projet et vous.
  • Gouverné par hooks exécutables. Les seuils (profondeur, budget de questions, plafond de branches) sont enforcés mécaniquement, pas par bonne volonté.
  • Multi-prismes cloisonnés. Les décisions structurantes passent au filtre de plusieurs angles de pensée indépendants, pour trancher sans biais ni angle mort.
  • Vérification de cohérence continue. Le moteur boucle automatiquement entre les notes voisines de votre dossier pour détecter les contradictions de chiffres, de noms, de dates et d'engagements.
  • Garde-fou anti-hallucination intégré. Les affirmations datées sont systématiquement converties en questions pointant vers des sources officielles vérifiables.
  • Reproductible, paramétrable, versionné. Chaque composant a une version, un historique, une date de revue programmée. Aucun composant n'est figé.
  • Mémoire des décisions. Chaque choix structurant est consigné, daté, retrouvable. Aucune décision ne se perd entre deux sessions de travail.
  • Cohérence des fichiers. Le moteur croise toutes les notes du dossier et détecte les contradictions — chiffres, noms, dates, engagements — avant qu'elles coûtent.
  • Exploration exhaustive. Avant d'affirmer, Octans explore les pistes non évidentes et balisé les zones incertaines avec une transparence totale.
  • Skills enchaînés. Une commande déclenche une séquence complète — audit, correction, rapport — sans intervention manuelle entre les étapes.
  • Conformité anticipée. RGPD, WCAG, CGV vérifiés en amont. Pas en urgence au go-live, quand il est trop tard pour changer d'architecture.
  • Sources vérifiées. Les affirmations datées sont converties en questions pointant vers des sources officielles. Octans n'affirme pas ce qu'il ne peut pas sourcer.
  • Versionnage natif. Chaque composant porte un numéro de version, une date de revue planifiée et un historique complet. Rien ne disparaît sans trace.
  • Pré-mortem automatique. Avant chaque décision clé, Octans simule les points de rupture potentiels et les signale avant qu'ils deviennent des coûts réels.
  • Actualisation continue. Skills et agents se mettent à jour. Votre CDC suit l'évolution de vos outils sans tout reprendre de zéro.
  • Séquençage priorisé. Octans cadre l'ordre des tâches selon l'impact et les dépendances réelles — pas selon l'urgence perçue ou l'habitude.
