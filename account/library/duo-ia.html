Vos 2 IA coordonnées — Claudian + Claude Code

vous installez 2 assistants IA qui partagent le même cerveau système. L'un organise votre vault Obsidian, l'autre exécute le code dans VSCode. Ensemble, ils gardent votre mémoire, traçent vos décisions et travaillent vite — sans se marcher dessus. Tout local chez vous.

Vous ne lances pas 2 IA isolées. Vous orchestres 2 IA qui partagent le même cerveau — un fichier de doctrine unique (CLAUDE.md) que vous écris une fois, qu'elles relisent à chaque session.

2 · Qui fait quoi
Claudian
Dans Obsidian · votre chef de bureau

Range vos documents, garde la mémoire du projet, suit les décisions verrouillées et propage les changements de doctrine dans toutes les notes liées.

  • Versionne vos notes (frontmatter + historique)
  • Crée et maintient les [[wikilinks]]
  • Capte vos décisions dans le journal append-only
  • Propage un renommage dans tout le vault
Claude Code
Dans VSCode · votre artisan

Exécute le travail concret avec les bons outils : édite le code, lance les scripts, fait les commits Git, déploie sur votre hébergement.

  • Édite HTML, CSS, Python, JS — directement
  • Lancez scripts (audits, tests, déploiement)
  • Crée des commits Git avec messages clairs
  • Patche plusieurs fichiers en lot
3 · Comment ils évitent de se marcher dessus
RègleCe que ça fait pour vous
Chacun sa zoneClaudian touche les .md (notes). Claude Code touche le reste (code, scripts, site). Aucun conflit Git, aucun écrasement.
Mode autonomeIls enchaînent les tâches réversibles sans demander confirmation à chaque étape. Vous gagnes du temps.
Gardez-fous automatiquesIls s'arrêtent automatiquement sur ce qui touche un prix, un nom commercial, ou une action irréversible. vous gardez la décision finale.
Traçabilité partagéeChaque action laisse une trace (version, historique, décisions). vous pouvez revenir 3 mois en arrière et savoir qui a fait quoi, quand, pourquoi.
4 · Comment leur parler

Vous écris des phrases naturelles. Voici les patterns qui marchent le mieux.

À Claudian (dans Obsidian) :

# organisation et propagation
« Crée une note sur [sujet] avec la structure canonique. »
« Capte cette décision dans DECISIONS et propage. »
« Renomme tous les "Foo" en "Bar" dans le vault. »
« Mets à jour le frontmatter de [note] avec [info]. »

À Claude Code (dans VSCode) :

# exécution et code
« Patche le menu de toutes mes pages HTML avec ce nouvel ordre. »
« Audite la cohérence des blocs partagés du site. »
« Crée un endpoint FastAPI pour la facturation Stripe. »
« Commit + push les changements en cours. »

Et vous pouvez enchaîner les 2 :

« Claudian, capte la décision X dans DECISIONS,
  puis dis à Claude Code de propager dans le site. »
5 · Et Claude Design ?

C'est une 3ᵉ IA complémentaire, spécialisée dans la création visuelle. vous pouvez l'utiliser en plus.

Réorganiser vos notes, capter décisions📖 Claudian

Règle d'or : modif ciblée = Code · Création visuelle = Design · Vault = Claudian. → Détails dans le module « Choisir le bon outil IA »

8 · Et si je veux une seule IA ?

Oui, vous pouvez. La méthode marche aussi en mono-IA.

🟢 Claude Code seul — vous perdez la lane Obsidian, mais vous pouvez toujours organiser vos notes manuellement.
🟢 Claudian seul — vous perdez la lane code, mais vous pouvez toujours coder à la main.
🟢 Les 2 ensemble ← recommandé : maximum de productivité, mémoire partagée, traçabilité totale.

💻 Machine modeste ? Vous pouvez aussi faire tourner le duo en deux instances de Claude Code — plus léger en CPU et en RAM qu'Obsidian + Claude Code.

Choisissez ce qui convient à votre flux. Le Pack vous donne les deux configurations prêtes à l'emploi.
